# 예제 2-8 설명
---
```
function a() {
    /* ... */
  } // 함수 선언문. 함수명 a가 곧 변수명.
  a(); // 실행 OK.
  
  var b = function() {
    /* ... */
  }; // (익명) 함수 표현식. 변수명 b가 곧 함수명.
  b(); // 실행 OK.
  
  var c = function d() {
    /* ... */
  }; // 기명 함수 표현식. 변수명은 c, 함수명은 d.
  c(); // 실행 OK.
  d(); // 에러!
```

# 예제 2-9 설명
---
```
function a() { ... } //함수 선언문, 이 경우 함수명 a가 변수명이 되고, 함수 a는 전역 스코프에서 접근 가능하다. 그래서 a();를 호출하여 실행할 수 있다.
var b = function() { ... }; // (익명) 함수 표현식, 함수가 변수 b에 할당되고, 변수명 b가 함수명이 된다. 
				   이 함수는 변수 b를 통해서만 접근할 수 있으므로, b();를 호출하여 실행할 수 있다.
var c = function d() { ... }; // 함수가 변수 c에 할당되고, 함수명은 d이다. 변수명은 c가 되고, 함수명은 d가 된다. 
				      이 함수 역시 변수 c를 통해서만 접근할 수 있으므로, c();를 호출하여 실행할 수 있다. 그러나 d();를 호출하면 에러가 발생한다. 
				      이는 함수 d는 변수 c의 스코프 내에서만 유효하기 때문이다. 함수 표현식에서 함수명은 해당 함수의 스코프 내에서만 유효하다.

※ 결과적으로:

a();, b();, c();는 모두 실행, 하지만 d();는 에러가 발생한다.
```
